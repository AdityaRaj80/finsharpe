## §0  PAPER POSITIONING (UPDATED)

The paper is a **trading-grade benchmark** with three coupled contributions:

1. **Benchmark.** Seven forecasters (4 modern + 3 classical) evaluated under identical, meticulously crafted cross-sectional trading strategies on a curated FNSPID universe. The methodological rigor (DSR, paired bootstrap, NW-HAC rank-IC, walk-forward CV, sliding window) **establishes the first reproducible end-to-end benchmark for trading-grade evaluation** of these architectures — most prior papers stop at MSE/MAE.

2. **Custom Sharpe + MDD Loss.** Backbone-agnostic *RiskAwareHead* wrapper + 5-term *CompositeRiskLoss* (differentiable Sharpe + heteroscedastic NLL + return-MSE + log-vol-MSE + profitability-supervised gate-BCE) — turns any forecaster into a trading-aware learner. Demonstrates **simpler architectures + correct loss can beat complex architectures + plain MSE**.

3. **Geometric identity (Theorem 1)** linking σ-prediction quality to Sharpe edge — closed-form regime predictor for when σ-aware sizing helps.

**Side-finding** (small contribution, baked into ablation table): **Global > Sequential** training mode for cross-sectional ranking. We have prior evidence on the corrupted dataset; re-verify on clean data using the 50-stock subset.

## §4  CALENDAR + WALK-FORWARD CV

Headline calendar:
```
Train:  2009-01-01 → 2021-12-31  (≈13 years, all with real sentiment)
Val:    2022-01-01 → 2022-12-31  (≈252 trading days, 280k news articles)
Test:   2023-01-01 → 2023-12-29  (≈251 trading days, 954k news articles)
```

**Walk-forward CV (robustness):** 4 expanding-window folds, each shifting by 1 year:

| Fold | Train end | Val | Test |
|------|-----------|-----|------|
| F1 | 2018-12 | 2019 | 2020 (incl. COVID stress) |
| F2 | 2019-12 | 2020 | 2021 |
| F3 | 2020-12 | 2021 | 2022 |
| **F4 (headline)** | **2021-12** | **2022** | **2023** |

We **report all 4 folds** in the paper, headline numbers from F4. Prior 3 folds are robustness evidence (Pesaran-Timmermann recursive style). This kills the "single-test-window p-hacking" critique flat.

**Sliding-window option:** for the H=20 and H=60 portfolios, also report a 252-day rolling-origin Sharpe series within F4 — shows performance stability through the year.

## §5  TARGET + LOSS — corrected from v1

**Target:** H-step ahead **log-return** = `log(close_{t+H} / close_t)`. Replaces the v1 MinMax-scaled-price target (which Jury D flagged as ❌ FAIL — incomparable to PatchTST/iTransformer/MASTER baselines).

**Loss function — KEEP existing CompositeRiskLoss** (the user explicitly confirmed):
```
L = α·L_SR_gated + β·L_NLL + γ·L_MSE_R + δ·L_VOL + η·L_GATE_BCE
```
Phase schedule: Phase 1 (warm-up MSE-only, ε=0..5) → Phase 2 (mixed, ε=5..15) → Phase 3 (Sharpe-dominant, ε=15+). Already implemented and tested (34/34 tests pass).

**Inference position function:** Sharpe-saturated `tanh(α·μ/σ) × gate`, leg-normalized.
